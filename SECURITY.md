## Reporting a Vulnerability

If you discover a security vulnerability within this project, **please do NOT open a public issue**.

Instead, please report it responsibly by:

1. **Email**: Send details to the repository maintainer via GitHub private messaging
2. **GitHub Security Advisories**: Use [GitHub's private vulnerability reporting](https://github.com/Masriyan/Claude-Code-CyberSecurity-Skill/security/advisories/new)

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if any)

### Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 7 days
- **Fix release**: Within 30 days for critical issues

## Responsible Use

This skill collection contains tools and scripts designed for **authorized security testing only**. Users must:

1. Obtain **proper written authorization** before testing any system
2. Comply with all **applicable laws and regulations**
3. Use tools only in **controlled, authorized environments**
4. **Never use** these tools for unauthorized access or malicious purposes
5. **Report vulnerabilities** found during authorized testing to the affected parties

## Scope

This security policy covers:

- The skill collection code and scripts
- Documentation and configuration templates
- Example configurations and templates

This policy does **NOT** cover:

- Third-party tools referenced by the skills
- Systems or networks tested using these skills
- User-generated content or modifications
